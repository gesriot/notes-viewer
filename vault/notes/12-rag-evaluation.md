# Оценка RAG-систем

## Что оценивается

RAG-конвейер оценивается на трёх уровнях:

```
Вопрос → [Выбор ретривера] → [Извлечение контекста] → [Генерация ответа]
              ↑                        ↑                        ↑
         Точность выбора         Полнота контекста         Качество ответа
```

## Метрики RAGAS

| Метрика | Что измеряет | Формула |
|---------|-------------|---------|
| **Context Recall** (Полнота контекста) | Доля релевантных документов, попавших в контекст | $\frac{\text{relevant in context}}{\text{total relevant}}$ |
| **Faithfulness** (Достоверность) | Насколько ответ опирается только на контекст | $\frac{\text{claims from context}}{\text{total claims}}$ |
| **Answer Correctness** (Корректность) | Совпадение ответа с эталоном | F1 по семантическому сходству |

## Что такое RAGAS

**RAGAS** (Retrieval-Augmented Generation Assessment) — фреймворк для автоматической оценки RAG-систем. Позволяет оценивать систему без ручной разметки ответов.

## Разработка бенчмаркового датасета

Шаги создания тестового набора:

1. **Составить тестовые примеры** — вопросы с эталонными ответами и источниками
2. **Разнообразие типов вопросов** — простые фактические + сложные аналитические
3. **Запустить оценку** — прогнать систему через все вопросы
4. **Анализ наблюдений** — где система ошибается и почему

## Оценка разных ретриверов

| Ретривер | Сильные стороны | Слабые стороны |
|----------|----------------|----------------|
| Векторный поиск | Семантически схожие документы | Не умеет фильтровать/агрегировать |
| Cypher-шаблоны | Точные структурированные запросы | Негибкие, нужно писать вручную |
| text2cypher | Гибкие запросы к графу | Может генерировать неверные запросы |
| GraphRAG (глобальный) | Тематические вопросы по всему корпусу | Высокое потребление токенов |
| GraphRAG (локальный) | Конкретные вопросы о сущностях | Требует предварительной индексации |

## Гибридный поиск как улучшение базового RAG

Комбинация **векторного поиска** (семантика) и **полнотекстового поиска** (ключевые слова):

$$
\text{score}(d) = \alpha \cdot \text{VectorSimilarity}(d) + (1-\alpha) \cdot \text{BM25}(d)
$$

Улучшает результаты для запросов с конкретными именами, терминами, аббревиатурами.

## Наблюдения из практики оценки

При оценке GraphRAG vs обычного RAG часто проявляются такие паттерны:
- **Глобальный поиск** сильнее на широких аналитических вопросах
- **Локальный поиск** точнее на вопросах о конкретных сущностях
- **Обычный RAG** достаточен для простых фактических вопросов
- Оба метода хуже справляются с вопросами, требующими числовой агрегации — там нужен text2cypher

## Типы вопросов для бенчмарка

Хороший бенчмарк должен включать разные типы:

| Тип | Пример | Лучший метод |
|-----|--------|-------------|
| Фактический | "Кто такой Улисс?" | Local GraphRAG |
| Тематический | "Какие основные темы в «Одиссее»?" | Global GraphRAG |
| Сравнительный | "Чем отличаются боги от людей в тексте?" | Global GraphRAG |
| Аggregation | "Сколько персонажей упомянуто?" | text2cypher |
| Multi-hop | "Кто наставник сына Агамемнона?" | Local GraphRAG |
