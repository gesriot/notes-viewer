# Text2Cypher: запросы к графу из естественного языка

## Зачем нужен text2cypher

Векторный поиск находит **семантически похожие** документы, но не умеет:
- Агрегировать данные (`AVG`, `COUNT`, `SUM`)
- Сортировать и фильтровать по условиям
- Строить многоуровневые запросы через граф

Пример запроса, на который нельзя ответить векторным поиском:

> "Перечислите три фильма с наибольшим рейтингом режиссёра Стивена Спилберга и их среднюю оценку"

Для этого нужен Cypher:
```cypher
MATCH (:Reviewer)-[r:REVIEWED]->(m:Movie)<-[:DIRECTED]-(:Director {name: 'Steven Spielberg'})
RETURN m.title, AVG(r.score) AS avg_rating
ORDER BY avg_rating DESC
LIMIT 3
```

## Место text2cypher в RAG-конвейере

```
Вопрос → [Получить схему графа] → [Сформировать промпт] → [LLM] → [Cypher] → [Neo4j] → Контекст → LLM → Ответ
```

Text2cypher используется как **ретривер** в более широком RAG-конвейере: вместо возврата текстовых чанков он возвращает точные структурированные данные из графа.

## Четыре техники повышения точности

### 1. Few-shot примеры в промпте

Показываем LLM несколько примеров «вопрос → Cypher»:

```
Вопрос: "Какой фильм снял Нолан в 2010?"
Cypher: MATCH (d:Director {name: 'Nolan'})-[:DIRECTED]->(m:Movie {year: 2010}) RETURN m.title

Вопрос: "Сколько фильмов в базе?"
Cypher: MATCH (m:Movie) RETURN count(m)
```

Few-shot помогает LLM понять синтаксис конкретной схемы и правила форматирования.

### 2. Схема базы данных в промпте

Без схемы LLM вынуждена угадывать имена узлов и свойств. Схема — это маппинг между семантикой вопроса и моделью графа:

```
Узлы: Movie (title, year), Person (name, born), Director
Связи: DIRECTED(Director → Movie), ACTED_IN(Person → Movie)
Свойства: Movie.rating (float), Person.born (int)
```

### 3. Маппинг терминологии

Пользователи используют разные слова для одних сущностей:

| Пользователь говорит | В графе |
|----------------------|---------|
| "режиссёр", "снял" | `Director`, `DIRECTED` |
| "актёр", "играл" | `Person`, `ACTED_IN` |
| "оценка", "рейтинг" | `Movie.rating` |

Добавление маппинга в промпт снижает ошибки в именах свойств и типах связей.

### 4. Инструкции по формату

Явно указать LLM:
- Возвращать только Cypher без объяснений
- Использовать `toLower()` для нечувствительного поиска
- Не использовать функции, которых нет в версии Neo4j

## Специализированные дообученные модели

Помимо GPT-4o, существуют модели, дообученные специально для text2cypher. Преимущества:
- Меньше ошибок в Cypher-синтаксисе
- Быстрее и дешевле для этой конкретной задачи

Недостаток: хуже на нестандартных схемах, требуют переобучения при смене версии Neo4j.

## Ограничения text2cypher

| Проблема | Причина | Решение |
|----------|---------|---------|
| Неверные имена свойств | LLM угадывает | Всегда предоставлять схему |
| Неправильный синтаксис | Разные версии Cypher | Few-shot + инструкции по версии |
| Сложные многоуровневые запросы | LLM теряет граф | Специализированные ретриверы |
| Несуществующие агрегации | Галлюцинации | Валидация запросов перед исполнением |

В агентном RAG: text2cypher используется как **универсальный ретривер-запасной вариант**, когда нет специализированного ретривера для конкретного вопроса.

## Cypher vs векторный поиск: когда что использовать

| Тип вопроса | Метод |
|-------------|-------|
| "Расскажи про Эйнштейна" | Векторный поиск |
| "Кто снял «Начало»?" | Text2Cypher |
| "Топ-5 фильмов по рейтингу" | Text2Cypher |
| "Найди статьи похожие на эту" | Векторный поиск |
| "Сколько контрактов с ACME активны?" | Text2Cypher |
| "Найди договоры об условиях оплаты" | Векторный поиск |
