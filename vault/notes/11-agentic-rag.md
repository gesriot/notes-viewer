# Агентный RAG (Agentic RAG)

## Что такое Agentic RAG?


**Агентный RAG** — система, в которой несколько специализированных агентов-ретриверов совместно отвечают на сложные вопросы. В отличие от обычного RAG с одним ретривером, здесь LLM выбирает нужный инструмент под конкретный запрос.

## Три компонента системы

$$
\text{UserQuery} \to \underbrace{\text{Retriever Router}}_{\text{tool selection}} \to \underbrace{\text{Retriever Agent}}_{\text{data retrieval}} \to \underbrace{\text{Answer Critic}}_{\text{critique}}
$$

### 1. Агенты-ретриверы (Retriever Agents)

Два типа:
- **Специализированный** — жёстко закодированный запрос к БД с параметрами (например, поиск фильма по названию)
- **Универсальный** — text2cypher для генерации Cypher-запроса из любого вопроса (резервный вариант)

Инструменты описываются в формате OpenAI tool definitions и добавляются по мере необходимости.

### 2. Маршрутизатор ретриверов (Retriever Router)

LLM получает список доступных инструментов и вопрос → выбирает нужный ретривер и параметры.

Вопросы обрабатываются **последовательно** — ответ на предыдущий вопрос используется для уточнения следующего:

$$
q_{i+1} = \text{LLM}(\text{rewrite}(q_i,\ \text{answers}_{1..i}))
$$

### 3. Критик ответов (Answer Critic)

После получения ответов от ретриверов LLM проверяет полноту:
- Если ответ **полный** → вернуть пользователю
- Если **неполный** → сгенерировать новые вопросы и пройти ещё один раунд поиска

Критика выполняется **один раз**: если после второго круга ответ всё ещё неполный — возвращается как есть с рекомендацией переформулировать вопрос.

## Когда нужен Agentic RAG?

| Ситуация | Решение |
|----------|---------|
| Несколько разных источников данных | Разные ретриверы для каждого |
| Сложные структурированные данные | Специализированный ретривер вместо универсального |
| Многошаговые вопросы | Последовательная обработка с обновлением запросов |
| Нужна проверка качества ответа | Критик ответов |

## Формула качества системы

$$
\text{Quality}_\text{agentic} = \frac{\text{Correct Answers}}{\text{Total Queries}} \cdot \frac{1}{1 + \text{Avg Rounds}}
$$

Цель — максимум правильных ответов при минимуме раундов поиска.

## text2cypher — генерация запросов к графу

Универсальный ретривер для структурированных данных — преобразует вопрос на естественном языке в Cypher-запрос к Neo4j.

Техники повышения точности:
- **Few-shot примеры** в промпте
- **Схема БД** в промпте (описание структуры графа)
- **Маппинг терминологии** (синонимы → поля графа)
- Специализированные дообученные модели text2cypher
