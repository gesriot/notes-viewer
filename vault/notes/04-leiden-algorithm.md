# Алгоритм Leiden для детекции сообществ

## Что такое Leiden?

**Leiden** — алгоритм иерархической кластеризации графов, используемый в GraphRAG
для группировки связанных сущностей.


![Leiden Algorithm](../images/leiden-algorithm.jpg)

## Шаги алгоритма

| Этап | Описание | Формула |
|------|----------|---------|
| 1. Инициализация | Каждый узел = отдельное сообщество | $c_i = i,\ \forall i \in V$ |
| 2. Локальное перемещение | Узлы переносятся в сообщества соседей | Максимизация $\Delta Q$ |
| 3. Агрегация | Сообщества становятся узлами | Создание мета-графа |
| 4. Рефинемент | Улучшение разбиения | Повторение шагов 1–2 |

## Функция модулярности

$$
Q = \frac{1}{2m} \sum_{c} \left[ e_c - \frac{K_c^2}{2m} \right]
$$

где:
- $m$ — общее число рёбер
- $e_c$ — число рёбер внутри сообщества $c$
- $K_c$ — сумма степеней узлов в сообществе $c$

## Преимущества Leiden над Louvain

1. **Гарантия связности** — все сообщества связны
2. **Быстрая сходимость** — $O(n \log n)$
3. **Масштабируемость** — работает с миллионами узлов

## Leiden vs Louvain в реализациях GraphRAG

| | Leiden | Louvain |
|-|--------|---------|
| **MS GraphRAG (статья)** | ✅ используется | — |
| **Практические реализации** | — | ✅ часто используется |
| **Гарантия связности** | ✅ все сообщества связны | ❌ могут быть несвязные |
| **Скорость** | $O(n \log n)$ | $O(n \log n)$ |

Оба алгоритма максимизируют модулярность — разница в надёжности разбиения.

## Структура сводки сообщества (LLM-отчёт)

После детекции LLM генерирует структурированный отчёт по каждому сообществу:

```
- ЗАГОЛОВОК: краткое название сообщества с ключевыми сущностями
- РЕЗЮМЕ: как сущности связаны и что важно
- РЕЙТИНГ ВОЗДЕЙСТВИЯ: 0–10 (серьёзность влияния сообщества)
- ОБЪЯСНЕНИЕ РЕЙТИНГА: одно предложение
- ПОДРОБНЫЕ ВЫВОДЫ: 5–10 ключевых инсайтов
```

## Применение в GraphRAG

$$
\text{Entities} \xrightarrow{\text{Louvain/Leiden}} \text{Communities} \xrightarrow{\text{LLM}} \text{Structured Reports}
$$

## Проблема суперузлов

При большом корпусе некоторые сущности (суперузлы) имеют непропорционально много описаний и связей. Это требует отдельной обработки при обобщении.
